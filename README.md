# 个人简历 · 品牌形象网页

黑色 / 深色系 + 渐变彩色点缀的设计师个人品牌展示页。**纯静态**（HTML + CSS + 原生 JS，零构建、零运行时依赖），双端自适应（PC 侧栏 / 移动汉堡），内置后台管理（`?admin`）可实时编辑全部 8 个简历模块，并支持生成访问二维码（PNG / SVG 下载，微信扫码自动适配手机端）。

---

## ✨ 功能一览

| 模块 | 说明 |
|------|------|
| 个人信息 | 姓名、头像（URL / 本地上传）、职位、技能标签、简介、联系方式（电话/邮箱/微信/社交） |
| 教育背景 | 学校、专业、学历、起止时间、荣誉、课程（多条，可排序） |
| 实习经历 | 公司、职位、起止时间、工作内容（多条，可排序） |
| 工作经历 | 公司、职位、起止时间、工作内容（多条，可排序） |
| 项目经历 | 名称、角色、起止时间、描述、技术栈、项目链接（多条，可排序） |
| 科研经历 | 名称、角色、起止时间、研究内容、论文/成果链接（多条，可排序） |
| 所获荣誉 | 名称、颁发机构、时间（多条，可排序） |
| 自我评价 | 富文本（加粗/斜体/标题/列表/引用/链接/图片） |

- **响应式双端**：PC（>1024px）左侧竖向导航；平板（769–1024px）顶部横向导航；手机（≤768px）汉堡菜单 + 单栏。CSS 媒体查询自动切换，扫码与电脑打开同一 URL。
- **后台实时编辑**：所有改动即时写入浏览器 `localStorage`，刷新即生效，无需重新部署。
- **富文本编辑**：内置轻量富文本（`contenteditable` + `execCommand`），支持加粗、斜体、标题、列表、引用、插入链接、插入图片（本地上传 base64 / URL）。
- **拖拽排序**：经历/项目/荣誉模块支持拖拽调整顺序。
- **JSON 导入 / 导出**：导出整包数据用于备份或迁移；导入 JSON 覆盖数据。
- **二维码**：点击右下角按钮生成当前页面 URL 的二维码，支持下载 PNG / SVG。微信扫码自动打开手机端布局。

## 🚀 本地预览

无需任何环境（不用 Node / npm / 构建）。任选其一：

**方式 A：直接双击**
直接双击 `index.html`，在浏览器打开即可。

**方式 B：本地静态服务器（推荐，二维码/相对路径更稳）**
```bash
# Python 3（系统自带）
cd resume-brand
python -m http.server 8080
# 然后访问 http://localhost:8080/

# 或 Node（若已装）
npx serve .
```

> ⚠️ 二维码下载（`canvas.toDataURL`）在 `file://` 协议下可能受限，建议用方式 B 跑本地服务器。

## 🔐 后台管理

### 访问入口
在网址后加 `?admin`：

```
http://localhost:8080/?admin          # 本地
https://你的域名/?admin               # 部署后
https://你的域名/子路径/?admin        # 子路径部署
```

### 默认密码
```
admin123
```
**首次登录后请立即在「修改密码」中改掉。** 密码以 SHA-256 哈希存于浏览器 `localStorage`（`rb:pwd`）。登录态仅存于当前浏览器标签页的 `sessionStorage`，关闭页面即失效。

### 编辑操作
- 进入编辑模式后，顶部出现工具栏：**导出 JSON / 导入 JSON / 修改密码 / 重置数据 / 退出编辑**。
- 每条记录卡片右上角（hover 显示，触屏常显）：**✎ 编辑 / 🗑 删除 / ⠿ 拖拽排序**。
- 每个模块底部有「＋ 新增记录」按钮。
- 个人信息（Hero）左上角有 ✎ 按钮编辑；自我评价区有「编辑自我评价」按钮。
- 所有改动**实时保存**并立即刷新展示页。

## 📦 部署到线上

由于是纯静态站点，任何静态托管均可。**把 `resume-brand` 目录内的全部文件上传到根目录（或子目录）即可。**

### GitHub Pages
1. 把项目推到 GitHub 仓库。
2. 仓库 **Settings → Pages**，Source 选 `main` 分支、`/ (root)` 目录，保存。
3. 等待 ~1 分钟，访问 `https://用户名.github.io/仓库名/`，后台地址为 `…/仓库名/?admin`。

### Vercel / Netlify
1. 导入 Git 仓库或拖拽 `resume-brand` 文件夹。
2. 无需构建命令、无需环境变量，直接部署。
3. 访问分配的域名，后台加 `?admin`。

### 对象存储 / CDN（七牛 / 阿里 OSS / 腾讯 COS）
1. 创建一个 Bucket，开启静态网站托管。
2. 上传全部文件（保持目录结构）。
3. 绑定域名（微信内打开需 **HTTPS** 且备案/已配置域名）。
4. 访问域名根路径，后台加 `?admin`。

### 自有服务器（Nginx）
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/resume-brand;   # 指向 resume-brand 目录
    index index.html;
    location / { try_files $uri $uri/ /index.html; }
}
```

## 📱 二维码测试（微信扫码）

1. 把站点部署到带 **HTTPS** 的可访问域名。
2. 电脑浏览器打开站点 → 点击右下角圆形二维码按钮。
3. 弹窗中可「下载 PNG」或「下载 SVG」保存二维码图片。
4. 用微信「扫一扫」扫描（或扫描屏幕上的二维码）→ 微信内打开同一 URL，前端自动适配为手机端单栏布局。
5. 电脑浏览器打开同一 URL → 显示 PC 端侧栏布局。

> 二维码编码的是去掉 `?admin` 的纯展示页 URL（`location.origin + location.pathname`），确保访客扫到的是干净的展示页。

## 🔄 数据持久化机制说明

本项目数据采用**两层存储**，理解后可灵活管理：

| 场景 | 行为 |
|------|------|
| 单设备编辑 | 改动存入**当前浏览器**的 `localStorage`（key: `rb:data:v1`），同浏览器刷新即时生效。 |
| 多设备 / 公开发布 | 后台点「导出 JSON」得到 `resume-data-YYYY-MM-DD.json`；把它**改名为 `data/sample.json` 或通过「导入 JSON」分发**。 |
| 跨设备同步 | 纯前端方案天然无法跨设备自动同步。如需"一处编辑、全员生效"，请把导出的 JSON 内容写回部署目录（见下方「让默认数据生效给所有人」）。 |

### 让默认数据生效给所有人（可选）
若希望所有访客首次打开就看到你的内容（而非种子示例），把导出的 JSON 内容**直接粘贴覆盖 `js/data.js` 里的 `RB.DEFAULT_DATA` 对象**，重新部署即可。这样未在本地编辑过的访客也会看到你的内容。

> 注：访客在自己浏览器里做过的编辑仍优先（存于其本地 `localStorage`）。这是纯前端方案的预期行为。

## 🛠 常见问题

**Q：微信扫码后还是显示 PC 端布局？**

A：1）**确认已部署最新版本**。本地 `file://` 协议下的改动无法被微信扫码访问到。请把更新后的文件**部署到线上 HTTPS 地址**再扫码。2）**微信缓存**。微信内置浏览器对静态资源缓存非常激进，即使服务端更新了文件，微信可能仍用旧缓存。解决方法：
   - 方法 A：在微信里长按链接 → **"复制链接"** → 粘贴到**手机系统浏览器**（Safari/Chrome）里打开
   - 方法 B：在链接后面手动加版本号参数，如 `?v=2` 或 `?t=123456`，强制微信重新加载
   - 方法 C：微信内 → "我" → "设置" → "通用" → "存储空间" → "缓存" → 清除后重试
   - 页面已内置 `no-cache` meta 和资源版本号（`?v=2`），部署后**首次**访问应正常

3）代码层面已做了三重防御（CSS 媒体查询 + JS `screen.width`/`innerWidth`/UA 检测 + `<head>` 内联早执行），在真实手机微信环境下**一定是移动端布局**。如果仍看到 PC 布局，99% 是缓存问题。

**Q：扫码后还是显示 PC 端布局？**
A：二维码编码的是去掉 `?admin` 的 URL。布局由屏幕宽度（CSS 媒体查询）决定，与 UA 无关——手机物理宽度自然触发移动端布局。若用电脑扫码后又用电脑打开，自然是 PC 布局，属正常。

**Q：编辑后别人看不到我的修改？**
A：`localStorage` 是每个浏览器独立的。要让所有人看到，请按上文「让默认数据生效给所有人」把内容写回 `js/data.js` 并重新部署，或部署带后端的版本。

**Q：忘记密码？**
A：纯前端，无法找回。在浏览器控制台执行 `localStorage.removeItem("rb:pwd")` 后刷新，会重置为默认密码 `admin123`。

**Q：图片 / 数据丢失？**
A：数据存在浏览器本地，清除浏览器数据会丢失。重要内容请用后台「导出 JSON」备份。
